1. Introduction

Welcome to the Dynamic Decision System ("DDS") Authentication Platform. DDS is committed to protecting the privacy of our developers ("Developers") and the end-users ("End-Users") of our Developers' applications.

This Privacy Policy explains how we collect, process, share, and protect information when you utilize the DDS Developer Portal, our software development kits (SDKs), and our APIs. By creating an account or integrating DDS, you agree to the collection and use of information in accordance with this policy.

2. Information We Collect

We collect information necessary to operate the DDS Authentication Platform and deliver cryptographically secure, push-based verification services.

A. Developer Account Information

When Developers register for a DDS account, we collect credentials and details including:

  • Full name and profile picture (via Google Authentication or Firebase Auth)
  • Email address (for account notices, invoices, and alerts)
  • Company or corporate entity name
  • Company website URL
  • Developer preferred timezone (for usage limit resets)

B. Application Information

To configure authentication integration, Developers create application profiles containing application names, metadata configurations, logotypes, environment specifications, and callback URLs.

C. API Credentials

We generate and store cryptographically secure API credentials including Application IDs, Public API Keys, and one-way hashed SHA-256 signatures of Secret API Keys. Plaintext Secret API Keys are never stored in our systems.

D. Authentication Logs

To verify identities, we process phone numbers and generate temporary verification request tokens. We record and store:

  • Hashed phone numbers (E.164 format)
  • Request timestamp, expiration duration, and completion latency
  • Authentication results (PENDING, APPROVED, REJECTED, or EXPIRED)
  • Device details (browser type, operating system) and IP addresses associated with requests

E. Usage Analytics & Billing Information

We compile aggregate requests metrics for usage billing. In partnership with Stripe, we collect billing profile identifiers, payment history records, outstanding balances, and default payment card details (brand, last 4 digits, card expiration month and year).

3. Third-Party Services

We leverage industry-standard third-party subprocessors to handle infrastructure operations:

  • Stripe: Payment processing, invoices, and credit card validation. Credit card details are sent directly to Stripe and never touch our servers.
  • Firebase Authentication: Identity provider for secure Developer Portal authentication.
  • GitHub: Hosting of software development repositories, SDK packages, and documentation updates.

4. Data Retention

Developer profile data is retained as long as the account remains active. You can delete your developer account at any time under Settings, which permanently purges all credentials, invoices, and apps.

Authentication logs containing end-user IP addresses and E.164 phone numbers are automatically deleted or anonymized after ninety (90) days.

5. Data Security

DDS utilizes robust encryption standards. All communication is encrypted in transit using Transport Layer Security (TLS 1.3). Database collections are encrypted at rest using AES-256. API keys and passwords are hashed using salt-fortified bcrypt algorithms.

6. Developer Obligations

Developers must keep their Secret API Keys secure. Sharing or exposing Secret API Keys inside frontend web browsers or public client-side repositories (like GitHub) is strictly prohibited. You must integrate DDS credentials using secure backend environment configurations.

7. End-User Rights

End-users whose phone numbers are verified via DDS have the right to request deletion or access to their logs. Developers must provide clear privacy disclosures to their users outlining the use of third-party authentication services.

8. GDPR & CCPA Compliance

DDS complies with the General Data Protection Regulation (GDPR) and the California Consumer Privacy Act (CCPA). We act as a Data Processor for end-user authentication logs, and a Data Controller for Developer account information.

9. International Transfers

To provide global phone verification, your data may be transferred to and maintained on servers located outside of your state, province, or country. DDS ensures all transfers adhere to legal transfer mechanism frameworks.

10. Children's Privacy

DDS services are directed to developers aged 18 and older. We do not knowingly collect personal information from individuals under the age of 13.

11. Policy Updates

We may update our Privacy Policy periodically. We will notify you of any material changes by posting the new policy on this page and updating the "Last Updated" timestamp.
